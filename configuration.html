--- layout: default title: Configuration ---

Prontotype's default configuration should probably be just fine for you. But if you do want to tweak any of the settings, you can do so in the YAML configuration file config.yml found in the root of your Prontotype install.

Here's a brief overview of the settings, what they do and their default values:

#==================
# index
# -----------------
# Set to an empty string if using mod_rewrite or the nginx equivalent
#==================
index: index.php
				
#==================
# cache_path
# -----------------
# Specify a full server path to a writeable directory to enable caching of templates and data
#==================
cache_path: 

#==================
# authenticate
# -----------------
# Provide a username and password pair to enable password-protection of the prototype
#==================
authenticate:
  username: 
  password:

#==================
# prefix
# -----------------
# The string to use when prefixing cookie names, classes etc.
#==================
prefix: prontotype-
  
#==================
# cookie_lifetime
# -----------------
# How long cookies set by prontotype should persist for.
#==================
cookie_lifetime: 604800 # 1 week

#==================
# triggers
# -----------------
# URL segments that trigger specific behaviour
#==================
triggers:
  login: _login
  logout: _logout
  data: _data
  assets: _assets
  shorturl: _p

#==================
# nice_names
# -----------------
# A few short text strings that should be used in auto-generated navigation macros
#==================
nice_names:
  index: Overview     # what to label the index pages as
  home: Home          # what to label the home page as

#==================
# data
# -----------------
# Tweak to match the style of different data files.
#==================
data:
  csv:
    headers: true
    delimiter: ','
    enclosure: '"'
    escape: '\'
    id_header: id

Note that deleting a setting from the config.yml file will cause it to revert to it's default.
