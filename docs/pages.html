Working with Pages

Pages are just files with a .html extension that reside in the /site/pages directory.

All requests for pages are routed into this directory, which means that (at it's simplest) when Prontotype sees a request for http://example.com/account/test it will look for a page that lives at /site/pages/account/test.html. If it doesn't find a page there it'll look for a file at /site/pages/account/test/index.html, and if it still doesn't find anything it'll display a 404 page.

So you can basically structure your pages directory just like you would a normal, basic, HTML site.

However… files in the pages directory are also parsed as Twig templates. That means you can use a whole bunch of template tags within them to give you access to data, include other files, inherit layouts and more.
