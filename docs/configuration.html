Prontotype's default configuration should probably be just fine for you. But if you do want to tweak any of the settings, you can do so in the YAML configuration file config.yml found in the root of your Prontotype install.

Here's a brief overview of the settings, what they do and their default values:

Name Description Default
login_trigger The query string key that will trigger a simulated log in. See the Users & Logins section for more information. login
logout_trigger The query string key that will trigger a simulated log out. See the Users & Logins section for more information. logout
prefix The string that will be used as a prefix to namespace cookie names prontotype-
cookie_lifetime How long the cookies set by Prontotype should persist for (in seconds). 604800
debug If set to true this will enable some helpful debugging tools in case of problems. See the Debugging section for more information. false
auto_reload true
caching true
charset utf-8
csv_headers true
csv_delimiter ,
csv_enclosure \"
csv_escape \
csv_id_header id
homepage_nicename Home
index_nicename Overview
short_url_trigger p
json_data_trigger __data
compiled_css_trigger __css
xhtml_tags false
extensions

Note that deleting a setting from the config.yml file will cause it to revert to it's default.
