Prontotype makes use of Twig to power it's templates and YAML as a way to store structured data in text files. It's highly recommended to familiarise yourself with them before starting as understanding them well will mean you can really get the most out of Prontotype.

Download and installation

  1. Download and unzip the latest version (or clone it from the Github repo).
  2. Move the files to the web root of your server
  3. Configure your web server of choice to route all requests though the `index.php` file (details for Apache and Nginx below)

Apache Setup

Simply change the filename of the supplied `htaccess.txt` file to `.htaccess` to route all requests through the index file. (Note this may make it an 'invisible' file, depending on your OS).

Nginx Setup

server
{
    listen 80;
    server_name prototype-name.dev;
    root /path/to/prontotype;

    location /
    {
        index index.php index.html;
		if (!-f $request_filename)
		{
			rewrite ^/(.*)$ /index.php?/$1 last;
		}
    }
}

The Prontotype directory structure

Before diving in and creating some pages, it's worthwhile getting aquainted with the directory structure Prototype uses.

  • app - The application code. You shouldn't need to touch this.
  • data - Contains files of structured data that will be made accessible to your templates.
  • extensions - Extensions allow you to run code to customisethe behaviour of the prototype.
  • structure - Contains the pages, layouts, and macros that define your prototype.
  • config.yml - A YAML file containing any required overrides to the default configuration.
  • htaccess.txt - Sample .htaccess configuration file. If using Apache, this will need renaming as described above.
  • index.php - The bootstrap file that all requests need be run through.
